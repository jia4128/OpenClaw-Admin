## 核心需求详细描述

### R-01: 多用户 + RBAC 权限体系

**背景与目标**:
- 当前系统仅支持单用户本地密码登录，所有用户等价超级管理员
- 企业级应用必须具备多用户支持和细粒度权限控制
- 降低安全风险，支持团队协作

**功能规格**:
1. **用户管理**
   - 支持创建/编辑/删除用户
   - 支持用户名、邮箱、密码管理
   - 支持用户状态启用/禁用

2. **角色管理**
   - 预置三种角色：管理员、操作员、只读
   - 管理员：所有权限
   - 操作员：可执行大部分操作，但不能删除系统配置
   - 只读：仅查看权限

3. **权限控制**
   - 敏感操作（删除、重置）需二次确认或仅管理员可执行
   - 前端路由守卫
   - 后端 API 权限校验

**验收标准**:
- 可创建多个用户账户
- 不同角色登录后看到的功能菜单不同
- 尝试越权操作时提示无权限
- 所有写操作都有权限校验

**工作量预估**: 8 人天
- 后端用户管理 API: 2 人天
- 后端 RBAC 权限体系: 3 人天
- 前端用户管理页面: 2 人天
- 前端权限守卫集成: 1 人天

---

### R-02: 通知中心 + 告警渠道

**背景与目标**:
- 系统异常（Gateway 断开、任务失败、Agent 崩溃）用户无法及时获知
- 需要统一的通知中心汇聚所有告警和重要事件
- 支持多种告警渠道推送

**功能规格**:
1. **通知中心**
   - 统一展示所有告警和通知
   - 支持按类型、时间、状态筛选
   - 支持标记已读/未读
   - 支持批量操作

2. **告警规则**
   - Gateway 断开告警
   - 任务执行失败告警
   - Agent 崩溃告警
   - Token 用量超阈值告警
   - 计划外支出告警

3. **告警渠道**
   - 飞书机器人
   - 钉钉机器人
   - 邮件
   - Webhook（通用集成）

**验收标准**:
- 系统异常时能生成告警记录
- 告警能推送到配置的渠道
- 通知中心能查看所有历史告警
- 支持告警规则配置

**工作量预估**: 6 人天
- 后端通知服务: 2 人天
- 告警规则引擎: 2 人天
- 前端通知中心页面: 1 人天
- 告警渠道集成: 1 人天

### R-03: Office 智能体工坊基础功能

**背景与目标**:
- 当前 Office 模块为空壳，用户点击后无响应
- 需要实现多 Agent 协作的基础链路
- 验证用户是否真的需要此功能，避免过度设计

**功能规格**:
1. **场景创建**
   - 创建多 Agent 协作场景
   - 配置参与 Agent 列表
   - 定义场景描述和目标

2. **任务委派**
   - 在主 Agent 界面委派任务给子 Agent
   - 显示任务执行状态
   - 展示子 Agent 执行结果

3. **Agent 间通信**
   - 基础消息传递机制
   - 任务状态同步
   - 结果汇总展示

**验收标准**:
- 能创建多 Agent 协作场景
- 能委派任务并查看执行过程
- 能查看最终执行结果
- 基础链路跑通即可，细节后续迭代

**工作量预估**: 5 人天
- 后端场景管理 API: 2 人天
- 任务委派机制: 1 人天
- 前端场景管理页面: 1 人天
- 前端任务执行界面: 1 人天

---

### R-04: MyWorld 虚拟公司基础功能

**背景与目标**:
- 当前 MyWorld 模块为空壳，仅有静态 UI
- 需要实现可视化办公场景的核心流程
- 支持角色移动和区域互动

**功能规格**:
1. **可视化场景**
   - 展示办公区域布局
   - 显示角色位置
   - 支持角色移动（点击目标位置）

2. **区域互动**
   - 不同区域触发不同功能
   - 会议室：启动会议
   - 工位：开始工作
   - 休息区：查看状态

3. **实时通信**
   - 角色位置同步
   - 区域互动事件广播

**验收标准**:
- 能显示办公场景和角色位置
- 角色能移动到指定位置
- 区域互动能触发对应功能
- 核心流程可用即可

**工作量预估**: 5 人天
- 后端位置管理服务: 2 人天
- 区域互动逻辑: 1 人天
- 前端场景渲染: 1 人天
- 实时位置同步: 1 人天
